## Prerequisites

Before you begin, make sure you have the following:

- **Python 3.9 or higher** — [Download Python](https://www.python.org/downloads/)
- **Git** — [Download Git](https://git-scm.com/downloads)
- **A Supabase project** — [Create one at supabase.com](https://supabase.com). You will need your project URL and API keys.

<Tip>
  You can find your Supabase URL and keys in the Supabase dashboard under **Project Settings → API**.
</Tip>

---

## Set up locally

<Steps>
  <Step title="Clone the repository">
    Clone the Vitacore repository to your local machine:

    ```bash
    git clone https://github.com/msgiraldo01-wq/VITACORE.git
    cd VITACORE
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    Using a virtual environment keeps project dependencies isolated.

    <CodeGroup>

    ```bash macOS / Linux
    python3 -m venv venv
    source venv/bin/activate
    ```

    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    ```

    </CodeGroup>

    Once activated, your terminal prompt will show `(venv)`.
  </Step>

  <Step title="Install dependencies">
    Install all required packages from `requirements.txt`:

    ```bash
    pip install -r requirements.txt
    ```

    This installs Flask, the Supabase Python client, and python-dotenv.
  </Step>

  <Step title="Create the .env file">
    Create a `.env` file in the project root. This file holds your environment-specific secrets and is never committed to version control.

    ```bash .env
    FLASK_SECRET_KEY=replace-with-a-random-secret-key
    SUPABASE_URL=https://your-project-ref.supabase.co
    SUPABASE_ANON_KEY=your-anon-key
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    SUPABASE_TABLE_HC_PROFESIONALES=hc_profesionales
    ```

    | Variable | Description |
    |---|---|
    | `FLASK_SECRET_KEY` | Secret key used to sign session cookies. Use any random string locally. |
    | `SUPABASE_URL` | The base URL of your Supabase project. |
    | `SUPABASE_ANON_KEY` | Public anon key for client-side Supabase access. |
    | `SUPABASE_SERVICE_ROLE_KEY` | Service role key for privileged server-side operations. |
    | `SUPABASE_TABLE_HC_PROFESIONALES` | Name of the professionals table. Defaults to `hc_profesionales`. |

    <Warning>
      Never commit your `.env` file to Git. Verify that `.env` appears in your `.gitignore` before pushing.
    </Warning>
  </Step>

  <Step title="Run the application">
    Start the Flask development server:

    ```bash
    python app.py
    ```

    You should see output similar to:

    ```
     * Running on http://127.0.0.1:5000
     * Debug mode: on
    ```

    Open [http://127.0.0.1:5000](http://127.0.0.1:5000) in your browser.

    <Note>
      The development server runs with `debug=True` by default. This enables auto-reloading on code changes and detailed error pages. Do not use this mode in production.
    </Note>
  </Step>

  <Step title="Log in for the first time">
    Vitacore redirects unauthenticated users to the login page automatically. Use the credentials of a user that already exists in your Supabase `auth.users` table.

    If your Supabase project is fresh, create your first user in the Supabase dashboard:

    1. Open the Supabase dashboard and navigate to **Authentication → Users**.
    2. Click **Add user** and create a user with an email and password.
    3. Return to Vitacore and log in with those credentials.

    After login, you are redirected to the clinical dashboard (`hc/dashboard.html`).

    <Tip>
      To control what a user can access, assign them a role through **Administration → Users** after logging in as an admin.
    </Tip>
  </Step>
</Steps>
