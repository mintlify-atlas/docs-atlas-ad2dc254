Vitacore registers ten Flask blueprints in `app.py`. Each blueprint owns a URL prefix and a set of routes. All routes except the auth endpoints require `@login_required`. Most routes also require `@rol_required`.

<Note>
Route tables below document the intended route surface based on the blueprint structure. Detailed per-endpoint behaviour is covered in the feature documentation for each area.
</Note>

## `bp_auth` — Authentication

**Variable:** `bp_auth` | **Prefix:** `/auth` | **Module:** `blueprints/auth/routes.py`

Handles user login and logout. These routes do not require `@login_required`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/auth/login` | Render the login form |
| `POST` | `/auth/login` | Submit credentials — validates username/password and writes the Flask session |
| `GET` | `/auth/logout` | Destroy the Flask session and redirect to `/auth/login` |

See [Login flow](/auth/login) for a detailed walkthrough.

## `bp_admin` — Admin tools

**Variable:** `bp_admin` | **Prefix:** `/admin` | **Module:** `blueprints/configuracion_roles/routes_admin.py`

Provides the route-sync utility that registers all Flask endpoints in the `rutas` table.

| Method | Path | Decorators | Description |
|---|---|---|---|
| `GET` | `/admin/sync-routes` | `@login_required`, `@rol_required` | Render sync confirmation page |
| `POST` | `/admin/sync-routes` | `@login_required`, `@rol_required` | Execute route sync — upserts all endpoints into `rutas` |

See [Route synchronization](/admin/route-sync) for details.

## `bp_permisos_rutas` — Route permissions

**Variable:** `bp_permisos_rutas` | **Prefix:** (configured in `routes_rutas.py`) | **Module:** `blueprints/configuracion_roles/routes_rutas.py`

Manages the mapping between routes and the permission matrix. The URL prefix for this blueprint is not specified in `app.py` and must be confirmed in `routes_rutas.py`.

<Warning>
The URL prefix for `bp_permisos_rutas` is not declared in `app.py` — it is defined inside `routes_rutas.py`. Check that file to confirm the prefix before constructing links to these routes.
</Warning>
